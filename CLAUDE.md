# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

A Docusaurus documentation site for Sui blockchain developer tutorials. Features multi-language support (English, Japanese, Korean), interactive live code examples with Sui wallet integration, and PTB (Programmable Transaction Block) builder demos.

## Commands

```bash
# Development
npm start               # Start dev server (English)
npm run start:ja        # Start dev server (Japanese)
npm run start:ko        # Start dev server (Korean)

# Build & Deploy
npm run build           # Build static site to ./build
npm run typecheck       # TypeScript type checking (runs in CI)
npm run serve           # Serve built site locally

# Docusaurus utilities
npm run clear           # Clear Docusaurus cache
npm run write-translations # Generate translation files
```

## Architecture

### Sui Integration
- **[src/theme/Root/index.tsx](src/theme/Root/index.tsx)**: App-level providers wrapping the entire site
  - `QueryClientProvider` → `SuiClientProvider` → `WalletProvider`
  - Default network: `devnet`
- **[src/theme/ReactLiveScope/index.tsx](src/theme/ReactLiveScope/index.tsx)**: Exposes Sui hooks (`useCurrentAccount`, `useSignAndExecuteTransaction`, `useSuiClient`, `useSuiClientQuery`) and `Transaction` class to live code blocks
- **[src/components/SuiWalletDock.tsx](src/components/SuiWalletDock.tsx)**: Wallet connect/disconnect UI component
- **[src/theme/NavbarItem/ComponentTypes.tsx](src/theme/NavbarItem/ComponentTypes.tsx)**: Custom navbar item for wallet display

### PTB Builder
- **[src/components/PtbBuilderDemo.tsx](src/components/PtbBuilderDemo.tsx)**: Interactive PTB builder component using `@zktx.io/ptb-builder`
- **[src/ptb/templates/](src/ptb/templates/)**: PTB template definitions for demos

### Content Structure
- **[docs/](docs/)**: English documentation (source of truth)
- **[i18n/ja/](i18n/ja/)**, **[i18n/ko/](i18n/ko/)**: Localized content following Docusaurus i18n structure
- **[specs/](specs/)**: Curriculum specifications and translation glossary
  - [specs/translation-glossary.md](specs/translation-glossary.md): Terminology consistency across languages

### Sidebar
Uses autogenerated sidebar from filesystem structure (`sidebars.ts`). Category labels are defined in `_category_.json` files.

## Writing Guidelines

When creating articles in `docs/`:
- Use a friendly, approachable tone for beginners (include encouraging phrases like "難しくありません", "It's not difficult", "어렵지 않습니다")
- Japanese: Use polite form (です・ます調)
- Korean: Use polite form (합니다/습니다체)
- English: Use friendly, conversational tone

## i18n Guidelines

When adding or updating content:
1. English docs in `docs/` are the source of truth
2. Translated content goes in `i18n/{locale}/docusaurus-plugin-content-docs/current/`
3. Use [specs/translation-glossary.md](specs/translation-glossary.md) for consistent terminology
4. UI strings (button labels, exact on-screen text) should be kept in original language
