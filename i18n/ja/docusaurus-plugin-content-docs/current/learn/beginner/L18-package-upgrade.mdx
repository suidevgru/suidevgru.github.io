# パッケージをアップグレードする

コントラクトを公開した後で「バグを直したい」「機能を追加したい」と思うことはよくあります。Suiでは **パッケージのアップグレード** という仕組みで、既存のパッケージを更新できます。手順を一つずつ見ていきましょう。

---

## UpgradeCap とは？

`sui client publish` でパッケージを公開すると、トランザクションの結果として **`UpgradeCap`** というオブジェクトが発行されます。

このオブジェクトが「アップグレードの鍵」です。

- 通常は、`UpgradeCap` を管理している主体だけがパッケージをアップグレードできます
- 誰かに譲渡することも、破棄することもできます（破棄するとアップグレード不可になります）
- アップグレードすると **新しい Package ID** が発行されます（古いIDのコントラクトはそのまま動き続けます）

まず自分のウォレットに `UpgradeCap` があることを確認しましょう。

以下の手順で UpgradeCap を確認します。

```bash
sui client active-address
```

現在のアクティブアドレスが表示されます。このアドレスがL16でpublishを実行したアドレスであることを確認してください。

```bash
sui client objects
```

現在のアクティブアドレスが所有するオブジェクト一覧の中から、`UpgradeCap` を探します。そのオブジェクトIDをメモしておいてください。

---

## アップグレードを実行する

コードの修正が終わったら、以下のコマンドでアップグレードします。

```bash
sui client upgrade --upgrade-capability <UPGRADE_CAP_ID>
```

`<UPGRADE_CAP_ID>` の部分は、先ほどメモした `UpgradeCap` のオブジェクトIDに置き換えてください。

コマンドが成功すると、トランザクションの結果に新しい Package ID が表示されます。

---

## 新しい Package ID を確認する

アップグレード後、Sui Explorer でトランザクションを確認しましょう。このコースは Devnet を使用しているので、[Devnet の Sui Explorer](https://suiexplorer.com/?network=devnet) を使います。

1. アップグレードのトランザクションハッシュで検索する
2. 「Object Changes」セクションを開く
3. `Published` として新しい Package ID が表示されていることを確認する

この新しい Package ID が、アップグレードされたパッケージのアドレスです。今後はこちらのIDを使ってコントラクトを呼び出します。

:::tip
古い Package ID のコントラクトは引き続き動作します。ただし、既存のクライアントコードや依存先は自動では新しい Package ID に切り替わりません。新しいバージョンを使いたい場合は、参照先の Package ID を更新する必要があります。
:::
