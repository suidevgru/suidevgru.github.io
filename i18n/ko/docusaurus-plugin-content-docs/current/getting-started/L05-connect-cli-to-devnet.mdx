# CLI를 Devnet에 연결하기

[Sui CLI 설치하기](./L03-install-sui-cli.mdx)에서 설치한 Sui CLI를 Devnet에 연결해 봅시다. **어렵지 않습니다**. 몇 가지 명령어만 실행하면 완료됩니다.

:::info
이 설정은 한 번만 하면 됩니다.
:::

## Sui CLI 초기 설정

Sui CLI를 처음 사용할 때는 초기 설정이 필요합니다. 다음 명령어를 실행하세요.

```bash
sui client envs
```

### 처음인 경우 (설정 파일이 없는 경우)

다음과 같은 메시지가 표시되면 초기 설정이 필요합니다.

```
No sui config found in `/home/user/.sui/sui_config/client.yaml`, create one [Y/n]?
```

**`y`**를 입력하고 Enter를 누르면 자동으로 키페어(주소)가 생성되고 설정이 완료됩니다.

```
y
Generated new keypair and alias for address with scheme "ed25519" [alias: 0x...]
  secret recovery phrase : [word1 word2 word3 ...]
Created "/home/user/.sui/sui_config/client.yaml"
Set active environment to testnet
╭─────────┬─────────────────────────────────────┬────────╮
│ alias   │ url                                 │ active │
├─────────┼─────────────────────────────────────┼────────┤
│ testnet │ https://fullnode.testnet.sui.io:443 │ *      │
│ mainnet │ https://fullnode.mainnet.sui.io:443 │        │
│ devnet  │ https://fullnode.devnet.sui.io:443  │        │
│ local   │ http://127.0.0.1:9000               │        │
╰─────────┴─────────────────────────────────────┴────────╯
```

:::danger 복구 구문을 안전하게 보관하세요
`secret recovery phrase`(복구 구문, 시드 구문이라고도 합니다)는 지갑의 "열쇠"입니다.

- 절대로 다른 사람에게 알려주지 마세요.
- 스크린샷을 찍거나, 채팅/이메일/클라우드 메모에 저장하지 마세요.
- 종이에 적어 오프라인으로 안전하게 보관해 주세요.

이 구문을 [Slush 지갑](./L01-install-slush.mdx)에 가져오면 CLI와 같은 주소를 브라우저 확장에서도 사용할 수 있습니다(선택 사항).
:::

<details>
<summary>Slush로 가져오는 방법 (선택 사항)</summary>

1. 브라우저 확장 프로그램 Slush Wallet을 엽니다 (요청 시 비밀번호 입력)
2. 오른쪽 하단의 **사용자 아이콘** → **Manage Account** → **Add account**를 엽니다
3. **Import passphrase**를 선택합니다
4. CLI의 복구 구문을 입력합니다

가져오기가 완료되면 Slush에서도 CLI와 같은 주소를 사용할 수 있습니다.

:::tip
필수는 아니지만, CLI로 만든 주소를 Slush로 가져오면 잔액과 주소를 확인하기 쉬워집니다.
:::

</details>

### 이미 설정된 경우

프롬프트 없이 네트워크 목록이 바로 표시되면 초기 설정이 이미 완료된 것입니다. 다음 섹션으로 진행하세요.

---

## Devnet에 연결하기

초기 설정 후 기본값은 **testnet**입니다. Devnet으로 전환해 봅시다.

```bash
sui client switch --env devnet
```

전환에 성공하면 다음과 같이 표시됩니다.

```
Active environment switched to [devnet]
```

제대로 전환되었는지 확인해 봅시다.

```bash
sui client active-env
```

`devnet`이 표시되면 성공입니다.

:::tip 네트워크 목록 확인하기
`sui client envs`로 설정된 네트워크 목록을 확인할 수 있습니다. `active` 열에 `*`가 표시된 것이 현재 네트워크입니다.

```
╭─────────┬─────────────────────────────────────┬────────╮
│ alias   │ url                                 │ active │
├─────────┼─────────────────────────────────────┼────────┤
│ testnet │ https://fullnode.testnet.sui.io:443 │        │
│ mainnet │ https://fullnode.mainnet.sui.io:443 │        │
│ devnet  │ https://fullnode.devnet.sui.io:443  │ *      │
│ local   │ http://127.0.0.1:9000               │        │
╰─────────┴─────────────────────────────────────┴────────╯
```
:::

## 주소 확인하기

CLI에 연결된 주소를 확인해 봅시다.

```bash
sui client active-address
```

`0x`로 시작하는 주소가 표시됩니다.

```
0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef
```

이 주소가 여러분의 Sui CLI 지갑 주소입니다.

---

## 성공 확인

다음을 확인할 수 있으면 성공입니다.

- `sui client active-env`가 `devnet`을 반환
- `sui client active-address`로 주소(`0x...`)가 표시됨
